    Replenishments¶

    Home > Replenishments

    Overview¶

    The Replenishments page lists all the replenishments that the system has calculated are necessary to complete the production.
    - The page displays Active replenishments by default, but the user can select to display Completed ones also.

    Column Details¶

    • #: This is an identifying number that refers to which case on which client pallet made the replenishment necessary. This number is clickable - see Viewing Replenishment Details below.
    • SKU: The product SKU
    • Name: The product description
    • Level: The InnoPick level (or level associated with some other subsystem if applicable) that the replen is assigned to.
    • Timeline: This graphic shows the progression of the replenishment at a glance.
      • It can be hovered over to reveal specific timestamps for each stage.
      • Click View full timeline to see more details.
      • The colours indicate whether the elapsed time for each step is within the typical range (Green, Yellow, Red).

    • Cases Depal: How many cases have been depalletized compared to the total amount for that replenishment.

    • Floor Position: Floor position code.

    • Layers Depal: # of layers to be depalletized.

    • HU: Handling Unit of the pallet.

    • InnoPick Replen. ID: Unique ID assigned to this replenishment by InnoPick.

      • This ID can be used to find the replenishment details in InnoPick Manager.

    • At the far right of each replenishment row is an icon that displays the system logs for this replenishment.

      • This is for system experts; normal operators do not need to know how to interpret logs.

    Depalletizer Pending Tasks Summary¶

    • There is a small "glasses" icon at the top of the Replenishments page. This can be pressed to open up the Depalletizer Pending Tasks Summary page:

    • Each Depalletizer's task details can be viewed by pressing the associated View button.

    Here are what each category means:

    • Requested to be Freed: Waiting on an operator to bring a pallet from floor location to PPB
    • Awaiting In Flight: A directed replen (from warehouse) has been called for and has not been delivered yet
    • Available on the Floor: The pallets corresponding to at least one replen task that the operator could in theory do.
      • Sometimes a single pallet can serve multiple replen tasks for one or more levels

    There is also the Preview Next Task Options button on the top right.

    • This button displays a detailed snapshot of the state of the depalletizing situation for a particular line.
    • The information can be copied using the button at the top of the popup and can then be shared with the support team for further investigation following an unusual observation or behavior from that depalletizing line.

    Inbound Accumulation¶

    In the top right section of the Replenishments page, the current Inbound Accumulation levels are displayed.

    The progress bars indicate how many cases are:

    1. On the Inbound Conveyor System between Descramblers and InnoPick infeed
    2. In Progress (the task has been "Started" by the Clamp operator)

    Note:
    - The levels have different amounts of possible accumulation, and that the amount of total cases can exceed the physical accumulation size.
    - This number decreases when cases are inducted into InnoPick or when an In Progress replenishment is shorted.

    Viewing Replenishment Details¶

    • The user can click on the # of the replen (left-most column) and see the case sequence of the pallet whose cases made the replenishment necessary.
    • Replenishments are only created when the system knows it will need a certain product in order to complete a client pallet. The case sequence for the pallet in question is displayed as a pop-up on the right-hand side of the screen without navigating the user away from the replenishments page.
    • For more details on the Pallet Case Sequence interface, see the Viewing the Case Sequence Section.

    Viewing Product Details¶

    • Users can also click the SKU Number and that will open up a similar pop-up with the product details.
      • See the section on the Product Details Sidebar.

    Replenishment Page Recovery Actions¶

    At the right-hand side of the Replenishments page, there is a circular icon that can be pressed to bring up the available Recovery Actions.

    Here is what each action does:

    • Short Replenishment Request: Pressing this will bring up a confirmation screen. If confirmed, this action "shorts" the replenishment in question. This tells the system that it will not receive these cases. MixMaster then recalculates whether a new replenishment for this product is needed.
    • Trigger Just In Time Out of Stock: This will declare a just in time out of stock process for this product, telling the system that it is not in fact available. MixMaster will then recalculate affected pallets.
    • Notify for last layer placed: This action notifies MixMaster (and afterwards the conveyor system) that the previous layer that was placed was in fact the last layer that will be placed. MixMaster then recalculates whether a new replenishment for this product is needed, and it updates the conveyor system.
    • Logs: This brings up any MixMaster logs associated with the selected replenishment. These are used for debugging by system experts, but advanced users can also use the information in them.
